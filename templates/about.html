Project Details

  • 5GB of data scraped from boardgamegeek.com
  • 4.4 million+ ratings
  • 115,000+ unique users
  • 70,000+ games
  • ~38 ratings/user games

How does it work?

We start with a big database of users and games. Users will have rated some games, but many of these entries will be missing. The recommender's job is to predict which scores the user would give the unrated games.



From the above ratings, we might say that User 1 likes any game related to Settlers of Catan, User 2 is into pirates, and User 3 is a fan of games made by Days of Wonder. It's impossible to say anything about User 4 because we have no useful ratings data. This is known as the cold start problem.


We should recommend games that match each user's specific preferences. Rather than trying to figure out which topics are of interest to our users, we use a machine Learning algorithm based on matrix factorization of the ratings data. This algorithm automatically discovers a small number of features which best capture the ratings data. Each user and game are weighted by how well these features represent them.



The learned features give us a compact way to represent user/game pairs. To find out which games we should recommend to a User 1, we vector-multiply the first row of the left matrix by each column from the right matrix. These products are our predictions for User 1's rating of each game. We now recommend the games whose predicted rating is the highest.
